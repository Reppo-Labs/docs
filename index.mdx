## How Reppo works

<Steps>
  <Step title="Choose your role">
    You can participate as a **publisher** (submit content to datanets), a **voter** (lock REPPO to curate quality), a **datanet owner** (create and manage a data market), or an **agent** (automate data pipelines via the API).
  </Step>
  <Step title="Authenticate">
    Platform API endpoints use a Privy session cookie. The Chat Server and Agent API use wallet-based auth: request a nonce, sign it with your wallet, and receive a 24-hour session token.
  </Step>
  <Step title="Publish or curate">
    Create a pod draft with `POST /me/pods`, then mint it on-chain on Base. Voters allocate veREPPO each 48-hour epoch to signal quality.
  </Step>
  <Step title="Earn emissions">
    High-performing pods earn REPPO emissions each epoch. Check claimable rewards and submit on-chain claim transactions through the API.
  </Step>
</Steps>
